# Todo 客户端需求规格

## 1. 项目概述

开发一款支持 Windows 和 macOS 的桌面端 Todo 应用，帮助用户高效管理日常任务。

## 2. 功能需求

### 2.1 任务管理（核心）
- 创建、编辑、删除任务
- 标记任务为完成/未完成
- 支持任务标题和详细描述
- 设置任务优先级（高/中/低/无）
- 设置截止日期和时间提醒
- 子任务（任务拆分）

### 2.2 分类与组织
- 清单/列表（list）：按场景分组（如"工作"、"个人"、"购物"）
- 标签（tag）：跨清单打标
- 收藏/星标重要任务
- 排序：按优先级、截止日期、创建时间、字母

### 2.3 视图
- 列表视图（默认）
- 看板视图（按状态分列）
- 今日视图（仅展示今日待办）
- 搜索与筛选

### 2.4 数据与同步
- 本地数据持久化（离线可用）
- 数据导入/导出（JSON/CSV）
- 备份与恢复

### 2.5 用户体验
- 快捷键支持
- 拖拽排序
- 撤销/重做
- 系统通知提醒
- 系统托盘常驻（可选）

## 3. 非功能需求

### 3.1 跨平台
- Windows 10+ 和 macOS 12+ 原生体验
- 统一代码库，平台差异最小化

### 3.2 性能
- 启动时间 < 2 秒
- 千级任务量下操作流畅
- 内存占用 < 200MB

### 3.3 可靠性
- 本地数据不丢失
- 崩溃后数据可恢复

### 3.4 国际化
- 支持中文和英文界面

## 4. 技术选型建议

| 方案 | 框架 | 优点 | 缺点 |
|------|------|------|------|
| A | Tauri + React/Vue | 体积小、性能好、Rust 后端 | 需 Rust 知识 |
| B | Electron + React/Vue | 生态成熟、开发快 | 体积大、内存高 |
| C | Flutter Desktop | 单一代码库、UI 一致 | 桌面端不够成熟 |

**推荐方案 B (Electron)**：生态成熟，快速交付，后续可迁移至 Tauri。
